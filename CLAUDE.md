# austincv.com — Project Context for Claude

Personal website for Austin Chungath Vincent. A static site with no build step, no framework, no dependencies beyond two Google Fonts.

## Project structure

```

├── css/

├── js/
│   └── main.js             # All JavaScript
```

## What the site does

A minimalist personal CV site with two distinct experiences:

**Desktop (>768px)**
- Fixed left panel: bold serif headline + role description + LinkedIn/GitHub links
- Right side: tall scroll container (100vh per role × 5 roles), sections generated dynamically from `roles.json`
- A sticky ID card in the centre-right — flips to reveal each role as you scroll
- Left panel copy fades out and in to match the current role
- Card gently swings and tilts with mouse movement when idle

**Mobile (≤768px)**
- Single-screen layout, no scroll — everything fits within `100dvh`
- Swipe left/right (or tap arrow buttons) to cycle through role cards
- Progress dots + role copy below the card
- LinkedIn/GitHub buttons at the bottom
- Portrait-only: landscape shows a "Please rotate your device" overlay

**Responsive switching** — when the viewport crosses the 768px breakpoint (e.g. browser resize or devtools), the active view is fully torn down and the correct one re-initialises from scratch.

## Key design decisions to preserve

- **Always light mode** — `color-scheme: only light` on `html` and `<meta name="color-scheme" content="only light">`. Never add dark mode.
- **Portrait-only on mobile** — landscape is blocked via CSS (`body { display: none }` + overlay at `max-width: 768px and orientation: landscape`).
- **No mirroring on card flip** — the flip uses a tilt-fade-out / content-swap / tilt-fade-in technique, not CSS `backface-visibility` or `preserve-3d`. The card always faces forward. Do not revert to a two-face approach.
- **Identical cards on desktop and mobile** — `buildCardInner()` is shared; there are no mobile-specific CSS overrides on card internals. Both views render the same card at the same dimensions.
- **Role sections are generated by JS** — `.role-section` divs are created dynamically in `desktopInit()` from the `roles` array. Do not add hardcoded `.role-section` divs to `index.html`.

## Roles data

All role content lives in `data/roles.json`. Each object has:

```json
{
  "company":  "string",   // shown on card label and mobile eyebrow
  "title":    "string",   // job title on card
  "dates":    "string",   // date range on card and mobile eyebrow
  "accent":   "#rrggbb",  // hex colour for card accent elements
  "icon":     "string",   // single char / emoji — fallback if logo fails to load
  "headline": "string",   // desktop left panel — use \n for line breaks
  "eyebrow":  "string",   // small uppercase label above headline
  "sub":      "string",   // one-sentence description below headline
  "headshot": "string",   // path to headshot image (e.g. assets/headshots/dojo.jpg)
  "logo":     "string"    // path to company logo PNG (e.g. assets/logos/dojo.png)
}
```

Roles are in reverse-chronological order (most recent first). Every card shows its `headshot` image with an `onerror` fallback to the `👤` placeholder emoji — there is no "photo only on first card" logic.

## Card dimensions

```css
--card-w: 260px
--card-h: 412px   /* ISO/IEC 7810 ID-1 portrait: 85.60×53.98mm */
```

Mobile cards use `height: min(var(--card-h), 56dvh)` with `aspect-ratio: 53.98 / 85.60` so the height is the primary constraint and width is derived. The headshot photo is `130px × 130px` on both desktop and mobile.

## Card flip mechanism (desktop)

`rotateTo(idx)` in `main.js` works in two phases:

1. **Tilt + fade out**: `perspective(600px) rotateY(±28deg) rotate(-1deg)` + `opacity: 0` over 0.2s
2. **Snap** to the opposite tilt (no transition), swap `cardFront.innerHTML`, then **tilt back in + fade in** to `rotateY(0deg) rotate(-2deg)` + `opacity: 1` over ~0.28s

Flip direction (±28deg exit side) is determined by comparing the new index to `prevRole` captured before `currentRole` is updated. An `animating` flag blocks new flips mid-animation.

## Mobile carousel

Cards are individually `position: absolute` inside `#mobileSwipeArea`, each centred at `top: 50%; left: 50%`. JS sets `translateX(calc(-50% + Xpx)) translateY(-50%)` where `X = (i - current) * slotW + dragOffset`.

- Normal navigation: `cubic-bezier(0.25, 1, 0.5, 1)` ease-out
- End-of-carousel spring-back: `cubic-bezier(0.34, 1.56, 0.64, 1)` overshoot
- Dragging past first/last card applies `× 0.18` rubber-band resistance
- Flick detection threshold: `flickVel > 3` (px/frame at 60fps)
- Drag commit threshold: `offsetX > slotW * 0.25`

## Responsive init / teardown

Both `desktopInit(roles)` and `mobileInit(roles)` return a `destroy()` function. The bootstrap uses `matchMedia('(max-width: 768px)')` and listens for `change` events:

```js
let destroy = mq.matches ? mobileInit(roles) : desktopInit(roles);
mq.addEventListener('change', e => {
  destroy();
  destroy = e.matches ? mobileInit(roles) : desktopInit(roles);
});
```

Each `destroy()` uses `AbortController.abort()` to remove all event listeners, cancels RAF / disconnects observers, and clears dynamically created DOM.

## Mobile layout constraints

The mobile layout must fit entirely within `100dvh` (no scroll):
- `.mobile` is `height: 100dvh; overflow: hidden`
- `.mobile-swipe-area` uses `flex: 1; min-height: 0` to fill remaining space
- Role copy is clamped to 2 lines with `-webkit-line-clamp: 2`
- Bottom links use `env(safe-area-inset-bottom)` for iPhone home indicator

Do not add `min-height` to any mobile flex children — it will break the no-scroll constraint.
